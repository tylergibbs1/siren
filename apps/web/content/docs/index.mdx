Siren uses typed JSON to describe diagrams. Write JSON in the editor, validate it, and render it live in the preview panel.

Siren does not invent a custom diagram language. JSON is the canonical format.

## Supported Diagram Types

| Type | Description |
| --- | --- |
| [Flowchart](/docs/flowchart) | Process flows, decision trees, pipelines |
| [Sequence](/docs/sequence) | API calls, service communication, request/response patterns |
| [Architecture](/docs/architecture) | Services, zones, and infrastructure diagrams |
| [ER](/docs/er) | Entities, fields, and relationships |
| [State](/docs/state) | State machines and transitions |
| [Class](/docs/class) | UML class diagrams with inheritance and associations |
| [C4](/docs/c4) | C4 model — people, systems, and boundaries |
| [Block](/docs/block) | Block diagrams with nested groups |
| [Requirement](/docs/requirement) | Requirements traceability and relationships |
| [Timeline](/docs/timeline) | Chronological events and milestones |
| [Mindmap](/docs/mindmap) | Hierarchical brainstorming and idea maps |
| [Git Graph](/docs/gitgraph) | Commits, branches, and merges |
| [Gantt](/docs/gantt) | Project schedules with tasks and sections |
| [Sankey](/docs/sankey) | Flow quantities between nodes |
| [Kanban](/docs/kanban) | Task boards with columns and cards |
| [Pie](/docs/pie) | Pie and donut charts |
| [Quadrant](/docs/quadrant) | 2×2 prioritization matrices |
| [Packet](/docs/packet) | Network protocol header bit layouts |

## Core workflow

The recommended Siren workflow is:

1. Create a Siren JSON document
2. Run [`validate()`](/docs/json-schema)
3. Render with [`fromJSON()`](/docs/render-from-json)

## Basic Structure

Every Siren diagram is a JSON object with a `type` field that determines the diagram kind:

```json
{
  "type": "flowchart",
  ...
}
```

```json
{
  "type": "sequence",
  ...
}
```

## Your First Diagram

Paste this into the [Playground](/) editor:

```json
{
  "type": "flowchart",
  "direction": "TB",
  "nodes": [
    { "id": "a", "label": "Start", "variant": "primary" },
    { "id": "b", "label": "Do something" },
    { "id": "c", "label": "Done", "variant": "success" }
  ],
  "edges": [
    { "from": "a", "to": "b" },
    { "from": "b", "to": "c" }
  ]
}
```

Nodes are the boxes. Edges are the arrows between them. Siren handles all the layout automatically.
